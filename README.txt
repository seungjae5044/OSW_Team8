This repository is for final project of the opensource software programing class.

Our Branch Rule:

1. [main] branch에는 직접 commit을 올리지 않는다.
2. 실습을 위한 룰
    2-1. 초기에 [main] branch 를 기준으로 팀원들은 각자의 실습을 위한 새로운 branch를 만든다.
    2-2. 이 브랜치의 이름은 [member/팀원이름] 형식으로 하고 수업이 끝날때 마다 commit을 한다.
    2-3. 매 수업마다 실습한 코드를 [main] branch 에다 merge 한다.
3. 로봇 프로그래밍을 위한 룰
    3-1. 하나의 로봇의 프로그래밍은 [main] branch를 기준으로 새로은 branch를 만들어 그곳에서만 한다.
    3-2. 이 branch의 이름은 [robots/robot#로봇번호#] 형식으로 한다.
    3-3. 기능 개발을 하기 전에 각 [robots/robot#] branch를 기준으로 새로운 branch를 만든다.
    3-2. 이 branch 이름은 [feature/기능이름] 형식으로 하고 한 명만 commit을 올린다.
    3-3. [feature/기능이름] branch에서 기능 개발이 끝나면 [robots/robot#] branch에 이를 merge 한다.
4. merge 할때 생기는 conflict들은 ~

These rules can be changed through discussion.
